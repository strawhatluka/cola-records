## Deviations from TRA Plan

### Deviation #1: `activeToolCalls` not in store
**TRA Plan:** Specified `activeToolCalls: Map<string, { toolName, toolInput }>` in T15 state shape
**Implementation:** Tool calls are tracked as messages in the `messages[]` array with `messageType: 'tool_use'`
**Assessment:** ACCEPTABLE. Storing tool calls as messages is simpler and provides the same functionality — the `tool_result` handler updates the matching tool_use message by scanning backward through messages. A separate Map would have been redundant.

### Deviation #2: `historyIndex` not in store
**TRA Plan:** Specified `historyIndex: number` in T15 state shape
**Implementation:** History index is local state in `ClaudeInputArea` component (`useState(-1)`)
**Assessment:** ACCEPTABLE. History navigation is a UI concern, not application state. Keeping it local to the input component follows React best practices and avoids unnecessary global re-renders.

### Deviation #3: ClaudeDiff component not implemented
**TRA Plan:** The Phase 5 overview mentioned "Diff viewer" but no task was assigned for it
**Implementation:** No ClaudeDiff component exists
**Assessment:** ACCEPTABLE. The TRA plan's 20-task breakdown (T1-T20) does not include a ClaudeDiff task. The overview text mentioned it speculatively, but it was implicitly descoped during task decomposition. It could be a follow-up WO if needed.

## Recommendations

1. **Test Coverage:** Consider adding dedicated unit tests for the 9 new leaf components (CodeBlock, ToolCall, Thinking, Spinner, SlashCommands, Permission, ContextBar, ConversationHistory, CostDisplay). These are currently tested only via integration mocks. This could be a follow-up WO.

2. **ClaudeDiff:** If inline diff display is still desired, create a follow-up WO. The infrastructure supports it — tool_result events from Edit operations could carry diff data.

3. **`onFileMention` prop:** ClaudeInputArea accepts `onFileMention` but ClaudePanel doesn't pass it. If @-mention file search is needed, an IPC handler for file tree querying would need to be wired up.
