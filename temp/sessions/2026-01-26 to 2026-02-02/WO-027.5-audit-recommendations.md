### Change Set 3: IPC Batching (Phase 2 — Performance)

**T-02: Batch `terminal:data` IPC events**
- [ ] Add `private dataBuffers = new Map<string, string[]>()` to TerminalService
- [ ] Add `private flushInterval: ReturnType<typeof setInterval> | null = null`
- [ ] In PTY `onData` callback (line 101): push data to buffer instead of sending
- [ ] Start 16ms flush interval on first session spawn:
  ```typescript
  private startFlushInterval() {
    if (this.flushInterval) return;
    this.flushInterval = setInterval(() => this.flushAllBuffers(), 16);
  }

  private flushAllBuffers() {
    for (const [sessionId, chunks] of this.dataBuffers) {
      if (chunks.length > 0) {
        this.mainWindow?.webContents.send('terminal:data', {
          sessionId,
          data: chunks.join(''),
        });
        this.dataBuffers.set(sessionId, []);
      }
    }
  }
  ```
- [ ] On session kill: flush remaining buffer, remove from Map
- [ ] On PTY exit: flush buffer before sending `terminal:exit`
- [ ] Clear interval when no sessions remain

**T-03: Debounce file watcher IPC events**
- [ ] Add pending events buffer to FileWatcherService:
  ```typescript
  private pendingEvents: { type: string; path: string }[] = [];
  private flushTimer: ReturnType<typeof setTimeout> | null = null;
  ```
- [ ] Replace direct `sendToRenderer` calls (lines 55, 62, 69, 76, 83) with buffer push
- [ ] Add 250ms debounced flush:
  ```typescript
  private queueEvent(type: string, filePath: string) {
    this.pendingEvents.push({ type, path: filePath });
    if (this.flushTimer) clearTimeout(this.flushTimer);
    this.flushTimer = setTimeout(() => this.flushEvents(), 250);
  }

  private flushEvents() {
    for (const evt of this.pendingEvents) {
      sendToRenderer(this.mainWindow, evt.type as any, evt.path);
    }
    this.pendingEvents = [];
    this.flushTimer = null;
  }
  ```
- [ ] No frontend changes needed — events still arrive individually, just batched in time
